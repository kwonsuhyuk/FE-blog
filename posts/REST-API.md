# REST API란 무엇인가?

REST API는 REST(Representational State Transfer) 아키텍처 스타일의 디자인 원칙을 준수하는 API를 의미합니다. 웹에서 자원을 효율적으로 관리하고 통신하기 위한 업계 표준으로 자리 잡았습니다.

## REST API 설계의 핵심 규칙

RESTful한 API를 설계하기 위해 반드시 지켜야 할 몇 가지 중요한 약속이 있습니다.

### 1. 자원의 식별 (URI)
- **URL 규칙**: 자원은 동사가 아닌 **명사**를 사용하며, 소문자로 작성합니다.
- **언더바(_) 대신 하이픈(-)**을 사용하며, 가독성을 위해 단어 사이를 구분합니다.
- **마지막에 슬래시(/)를 포함하지 않습니다.**
- **파일 확장자**(`.html`, `.json` 등)를 URI에 포함하지 않습니다.

### 2. 행위의 표현 (HTTP Method)
자원에 대한 행위는 URI에 직접 명시하지 않고 HTTP Method를 통해 나타냅니다.

- **잘못된 예**: `GET /getProducts` (행위 포함)
- **옳은 예**: `GET /products` (명사만 사용)

### 3. 일관된 명명 규칙
- 자원이 여러 개일 때는 **복수형**을 사용하는 것이 일반적입니다.
  - 예: `POST /products` (상품 등록)
  - 예: `GET /products/1` (1번 상품 상세 조회)

## REST API의 주요 이점

1. **가독성과 유지보수성**: URI만 보고도 어떤 자원에 접근하는지 직관적으로 알 수 있습니다.
2. **Stateless(무상태성)**: 서버는 클라이언트의 상태를 보관하지 않아 서버의 부담이 적고 확장이 용이합니다.
3. **독립성**: 클라이언트와 서버가 독립적으로 개발될 수 있습니다.

---

REST API는 단순히 주소를 만드는 규칙이 아니라, 웹 서비스의 확장성과 상호운용성을 높이기 위한 설계 철학입니다. 올바른 규칙을 준수하여 명확한 인터페이스를 제공하는 것이 중요합니다.
