# 커스텀 블로그를 구축하며 마주한 기술적 도전

기존의 블로그 플랫폼들은 편리하지만, 개발자가 원하는 미세한 UX 디테일과 코드 레벨의 최적화를 달성하기엔 명확한 한계가 존재합니다. 이번 프로젝트는 **Next.js 16**과 **Tailwind CSS v4**를 기반으로, AI와 실시간으로 설계 의도를 조율하는 **'Vibe Coding'** 방식을 통해 나만의 기술 아카이브를 구축하는 과정이었습니다.

단순한 구현을 넘어, 성능 최적화와 가독성, 그리고 유지보수가 용이한 아키텍처를 설계하며 고민했던 지점들을 기록합니다.

## 1. 아키텍처 설계: Next.js 16과 정적 최적화

Next.js 16의 App Router를 선택한 가장 큰 이유는 **서버 구성 요소(RSC)**를 통한 클라이언트 사이드 자바스크립트의 혁신적인 절감입니다. 

### Static Export와 성능 지표
GitHub Pages라는 제한된 환경에서 최상의 퍼포먼스를 내기 위해 `output: 'export'` 전략을 채택했습니다. 이는 빌드 타임에 모든 마크다운 데이터를 정적 HTML과 최소한의 JSON 페이로드로 변환합니다. 
결과적으로 브라우저는 첫 로딩 시 서버의 연산을 기다리지 않고 즉시 렌더링된 결과물을 받게 되며, 이후 페이지 전환 시에도 `next/link`를 통한 프리페칭(Prefetching) 덕분에 0.1초 내외의 지연 없는 이동이 가능해졌습니다.

## 2. Vibe Coding: 생산성 패러다임의 변화

'Vibe Coding'은 단순한 자동 완성을 넘어, 개발자가 요구하는 **구조적 의도(Structural Intent)**를 AI가 즉각적으로 이해하고 아키텍처로 풀어내는 방식입니다.

### 의도 중심의 리팩토링
초기 단계에서는 기능 검증을 위해 `PostsClient`라는 거대한 컴포넌트에 모든 로직이 포함되어 있었습니다. 이때 "UI의 독립성을 확보하고 확장성을 고려해 컴포넌트를 분리해줘"라는 의도를 전달했고, AI는 이를 바탕으로 `PostCard`, `Hero`, `TOC`, `CodeBlockManager` 등으로 관심사를 분리했습니다. 
이 과정에서 각 컴포넌트의 책임을 명확히 정의함으로써, 훗날 새로운 UI 요소를 추가하더라도 기존 비즈니스 로직을 건드리지 않는 **느슨한 결합(Loose Coupling)** 구조를 완성할 수 있었습니다.

### 동적 언어 처리의 자동화
히어로 섹션의 타이핑 애니메이션 중 마주한 한글 조사 처리 문제는 Vibe Coding의 효율성을 보여주는 좋은 사례입니다. 단어마다 '을/를'이 바뀌어야 하는 복잡한 조건을 유니코드 포인트 연산을 통해 해결했습니다.

```typescript
/**
 * 음절의 끝 글자에 받침이 있는지 판별하여 조사를 선택합니다.
 * 한글 유니코드 공식: (음절 코드 포인트 - 0xAC00) % 28
 * 결과값이 0보다 크면 종성(받침)이 있는 음절입니다.
 */
const getPostposition = (word: string) => {
  if (!word) return '';
  const lastChar = word.charCodeAt(word.length - 1);
  const isHangul = lastChar >= 0xac00 && lastChar <= 0xd7a3;
  
  if (!isHangul) return '를'; 
  
  const hasBatchim = (lastChar - 0xac00) % 28 > 0;
  return hasBatchim ? '을' : '를';
};
```

직접 로직을 설계하고 테스트하는 수고를 AI와의 대화로 대체하면서도, 엔지니어링 관점에서의 정확성은 그대로 유지하며 개발 속도를 3배 이상 높였습니다.

## 3. 디자인 시스템: 정보 밀도와 눈의 편안함

디자인의 핵심 원칙은 **선의 제거**와 **무채색 기반의 계층화**였습니다.

### 가독성을 위한 컬러 팔레트 설계
다크모드 구현 시 가장 경계한 것은 파란색 기운이 도는 피로한 배경색이었습니다. 이를 위해 중립적인 다크 그레이(`#171717`)를 배경으로 설정하고, 텍스트의 명도 대비를 4단계로 세분화하여 정보의 중요도에 따라 시각적 무게감을 조절했습니다. 
실선을 제거한 자리는 정교한 여백(`padding`, `margin`)과 폰트 웨이트의 대비가 채우게 하여, 화면이 복잡해지지 않으면서도 정보의 경계가 명확히 느껴지는 인터페이스를 지향했습니다.

### 폰트 로딩과 레이아웃 흔들림(CLS) 방지
`Gmarket Sans`와 `Pretendard`를 `next/font/local`로 직접 내장했습니다. 외부 CDN을 거치지 않으므로 네트워크 지연으로 인한 폰트 깜빡임(FOIT)을 원천 차단했고, 폰트 변수를 `:root` 레벨에서 관리하여 브라우저 확장 프로그램의 간섭 속에서도 일관된 타이포그래피를 유지하도록 강제했습니다.

## 4. 콘텐츠 경험 고도화

### 마크다운 처리와 문법 강조
`unified`와 `rehype-highlight`를 활용한 마크다운 파이프라인은 블로그의 본질인 '읽는 경험'을 강화합니다. 텍스트를 파싱하여 목차(TOC)를 생성하고, 코드 블록에는 사용자 편의를 위한 복사 버튼과 언어별 하이라이팅을 적용하여 기술 블로그로서의 전문성을 높였습니다.

## 5. 마치며: 성과와 향후 과제

Vibe Coding을 통한 이번 프로젝트는 **생산성 (Productivity)**과 **품질 (Quality)**이라는 두 마리 토끼를 잡은 성공적인 실험이었습니다. 

- **생산성**: 전체 빌드 프로세스 구축 시간을 70% 단축했습니다.
- **품질**: SEO 메타데이터 자동 생성, 스켈레톤 UI, 동적 목차 등 고차원적인 디테일을 설계 초기부터 안정적으로 포함했습니다.

AI는 이제 도구를 넘어 개발자의 사고 과정을 확장해 주는 파트너입니다. 이 공간이 저의 성장과 더불어 많은 동료 개발자들에게도 영감을 줄 수 있는 기록의 장이 되길 바랍니다.
