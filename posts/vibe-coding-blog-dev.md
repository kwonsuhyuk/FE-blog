# 커스텀 블로그를 구축하며 마주한 기술적 도전

기존의 블로그 플랫폼들은 편리하긴 하지만, 개발자가 원하는 미세한 UX 디테일이나 코드 레벨의 최적화를 달성하기엔 명확한 한계가 존재합니다. 

![Building Something New](https://images.unsplash.com/photo-1461749280684-dccba630e2f6?auto=format&fit=crop&w=1200&q=80)

이번 프로젝트는 **Next.js 16**과 **Tailwind CSS v4**를 기반으로, AI와 실시간으로 설계 의도를 조율하는 **'Vibe Coding'** 방식을 통해 나만의 기술 아카이브를 구축하는 과정이었습니다.

단순한 구현을 넘어 성능 최적화와 가독성, 그리고 유지보수가 용이한 아키텍처를 설계하며 고민했던 지점들을 살펴보겠습니다.

## 1. Vibe Coding: 생산성 패러다임의 변화

'Vibe Coding'은 단순한 자동 완성을 넘어, 개발자가 요구하는 **구조적 의도 (Structural Intent)**를 AI가 즉각적으로 이해하고 아키텍처로 풀어내는 방식입니다.

### 의도 중심의 리팩토링
초기 단계에서는 기능 검증을 위해 `PostsClient`라는 거대한 컴포넌트에 모든 로직이 포함되어 있었습니다. 이때 "UI의 독립성을 확보하고 확장성을 고려해 컴포넌트를 분리해줘"라는 의도를 전달했습니다.

AI는 이를 바탕으로 `PostCard`, `Hero`, `TOC`, `CodeBlockManager` 등으로 관심사를 분리했습니다. 이 과정에서 각 컴포넌트의 책임을 명확히 정의함으로써, 새로운 UI 요소를 추가하더라도 기존 비즈니스 로직을 건드리지 않는 **느슨한 결합 (Loose Coupling)** 구조를 완성할 수 있었습니다.

### 로직 구현의 효율화
히어로 섹션의 타이핑 애니메이션 중 마주한 한글 조사 처리 문제는 Vibe Coding의 효율성을 잘 보여주는 사례입니다. 단어마다 '을/를'이 바뀌어야 하는 복잡한 조건을 유니코드 포인트 연산을 통해 해결했습니다.

```typescript
/**
 * 음절의 끝 글자에 받침이 있는지 판별하여 조사를 선택합니다.
 * 한글 유니코드 공식: (음절 코드 포인트 - 0xAC00) % 28
 * 결과값이 0보다 크면 종성(받침)이 있는 음절입니다.
 */
const getPostposition = (word: string) => {
  if (!word) return '';
  const lastChar = word.charCodeAt(word.length - 1);
  const isHangul = lastChar >= 0xac00 && lastChar <= 0xd7a3;
  
  if (!isHangul) return '를'; 
  
  const hasBatchim = (lastChar - 0xac00) % 28 > 0;
  return hasBatchim ? '을' : '를';
};
```

이런 유틸리티성 로직 구현 시간을 단축하면서, 엔지니어는 전체적인 데이터 흐름이나 시스템 설계 같은 고차원적인 고민에 더 많은 에너지를 쏟을 수 있었습니다.

## 2. 디자인 시스템: 정보 밀도와 가독성

디자인의 핵심 원칙은 선의 제거와 무채색 기반의 계층화였습니다.

### 컬러 팔레트와 여백 설계
다크모드 구현 시 가장 경계한 것은 눈이 피로한 파란 기운이 도는 배경색이었습니다. 이를 위해 중립적인 다크 그레이(`#171717`)를 배경으로 설정하고, 텍스트 명도를 4단계로 세분화하여 정보의 중요도에 따라 시각적 무게감을 조절했습니다.

실선을 제거한 자리는 정교한 여백(`padding`, `margin`)과 폰트 웨이트의 대비가 채우게 하여, 화면이 복잡해지지 않으면서도 정보의 경계가 명확히 느껴지는 인터페이스를 지향했습니다.

### 폰트 로딩 최적화
`Gmarket Sans`와 `Pretendard`를 `next/font/local`로 직접 내장했습니다. 외부 CDN을 거치지 않으므로 네트워크 지연으로 인한 폰트 깜빡임(FOIT)을 원천 차단했고, 폰트 변수를 `:root` 레벨에서 관리하여 브라우저 확장 프로그램의 간섭 속에서도 일관된 타이포그래피를 유지하도록 강제했습니다.

## 3. 마치며: 성과와 느낀 점

Vibe Coding을 통한 이번 프로젝트는 **생산성 (Productivity)**과 **품질 (Quality)** 사이의 접점을 찾는 실험이었습니다.

- **생산성**: 전체 환경 구축 및 파이프라인 설계 시간을 기존 방식 대비 약 70% 단축했습니다.
- **품질**: 단순 구현에 그치지 않고 SEO 메타데이터 자동 생성, 스켈레톤 UI, 동적 목차 등 디테일한 요소들을 설계 초기부터 안정적으로 포함시켰습니다.

결론적으로 AI는 도구를 넘어 개발자의 사고 과정을 확장해 주는 파트너 역할을 수행합니다. 개발자가 로직의 '세부 구현'보다 '설계 의도'에 더 집중하게 함으로써 전체적인 프로젝트 퀄리티 상승으로 이어지는 구조를 확인할 수 있었습니다.
