# 안녕하세요! 권수혁입니다.

프론트엔드 개발자라면 누구나 한 번쯤 "나만의 멋진 블로그를 직접 만들고 싶다"는 꿈을 가집니다. 저 또한 그랬고, 오늘 그 결실인 **suhyukLog**를 세상에 선보이게 되었습니다. 

하지만 이번 개발 과정은 조금 특별했습니다. 단순히 코드를 한 줄 한 줄 직접 타이핑하는 고전적인 방식이 아니라, AI와 실시간으로 대화하며 영감을 주고받는 이른바 **'Vibe Coding'** 방식으로 탄생했기 때문입니다.

## 🛠 사용된 기술 스택

블로그의 뼈대와 살을 구성하는 기술들은 현재 프론트엔드 생태계에서 가장 진보된 도구들을 선택했습니다.

- **Framework**: [Next.js 15 (App Router)](https://nextjs.org/) - 최고의 퍼포먼스와 SEO를 위해.
- **Styling**: [Tailwind CSS v4](https://tailwindcss.com/) - 더 빠르고 직관적인 스타일링 엔진.
- **Animation**: [Framer Motion](https://www.framer.com/motion/) - 부드러운 인터랙션과 생동감 있는 UI를 위해.
- **Content**: Markdown (with gray-matter, remark) - 오직 글쓰기에만 집중할 수 있는 환경.
- **Typography**: Gmarket Sans & Pretendard - 미학적 완성도와 가독성의 조화.

## 🎨 'Vibe Coding'으로 구현한 디테일들

단순히 기능이 동작하는 것을 넘어, 제가 원하는 '감성'과 '사용자 경험'을 AI와 조율하며 만들어낸 핵심 디테일들입니다.

### 1. 타이핑 애니메이션 히어로 섹션
첫 화면에서 방문자를 맞이하는 "더 나은 [코드/경험/기록]을 위해 노력하는" 문구는 한 글자씩 써지는 타이핑 효과와 깜빡이는 커서가 적용되어 있습니다. 단순히 텍스트를 보여주는 것보다 훨씬 몰입감 있는 첫인상을 줍니다.

### 2. 스크롤 스파이 목차 (TOC)
긴 기술 아티클을 읽을 때 독자가 길을 잃지 않도록 우측에 목차를 배치했습니다. 현재 읽고 있는 위치를 시각적으로 보여주며, 원하는 섹션으로 즉시 이동할 수 있는 편의성을 제공합니다.

### 3. 완벽한 다크모드와 무채색 미니멀리즘
파란기가 섞이지 않은 정갈한 무채색 계열(`neutral-950`)의 다크모드를 구현했습니다. 배경 그라데이션을 과감히 걷어내고 텍스트와 여백의 힘만으로 고급스러운 분위기를 연출했습니다.

### 4. 성능 최적화 (Local Fonts & Static Export)
Gmarket Sans 폰트를 `next/font/local`로 직접 내장하여 로딩 속도를 극대화했습니다. 또한 GitHub Pages에 최적화된 정적 내보내기(Static Export) 방식을 사용하여 어디서든 빠르게 접속할 수 있습니다.

## 💻 코드의 흔적들

개발 과정에서 가장 중요하게 생각한 것은 **'리팩토링'**이었습니다. AI와 대화하며 초기에 뭉쳐있던 로직들을 `PostCard`, `Hero`, `TOC` 등의 재사용 가능한 컴포넌트로 분리하고, `useMemo` 등을 활용해 필터링 로직을 최적화했습니다.

```typescript
// 예: 동적 조사(을/를) 선택 로직
const getPostposition = (word: string) => {
  if (!word) return '';
  const lastChar = word.charCodeAt(word.length - 1);
  const isHangul = lastChar >= 0xac00 && lastChar <= 0xd7a3;
  if (!isHangul) return '를';
  const hasBatchim = (lastChar - 0xac00) % 28 > 0;
  return hasBatchim ? '을' : '를';
};
```

이런 작은 디테일 하나하나가 모여 블로그의 완성도를 결정짓는다는 것을 다시 한번 깨달았습니다.
