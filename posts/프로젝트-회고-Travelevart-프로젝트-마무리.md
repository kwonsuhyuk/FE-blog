2024.07 ~ 2024.08 약 한달 반 가량 진행한 여행지 추천 공유 서비스 **Travelevart** 프로젝트에 대해 회고하려고 한다.

## Keep: 잘했던 점

### 1. 성능 및 SEO 최적화
`next/image`와 `next/font`를 적극 활용하여 이미지와 폰트 로딩 성능을 개선했습니다. 특히 Lighthouse 성능 점수를 68점에서 91점까지 끌어올린 점이 뿌듯했습니다.

### 2. React Query 캐싱과 무한 스크롤
Tanstack Query를 사용해 서버 데이터를 캐싱하고, 무한 스크롤을 도입하여 대량의 데이터를 효율적으로 렌더링했습니다.

### 3. Axios 인터셉터 활용
공통된 인증 로직과 에러 처리를 인터셉터에서 관리하여 코드 중복을 획기적으로 줄였습니다.

## Problem: 아쉬웠던 점
- **디자이너의 부재**: 개발자들끼리 디자인을 하다 보니 UI/UX 디테일에서 아쉬움이 남았습니다.
- **Server/Client 컴포넌트 활용 미숙**: 초기 설계 시 모든 컴포넌트에 state를 사용하다 보니 서버 컴포넌트의 이점을 충분히 살리지 못했습니다.

## Try: 다음 프로젝트에서 시도해볼 것
- **Storybook 도입**: UI 컴포넌트를 독립적으로 테스트하고 개발하는 TDD 방식을 경험해보고 싶습니다.
- **TDD (Test Driven Development)**: 코드의 안정성을 위해 테스트 코드 작성을 습관화할 예정입니다.

이번 프로젝트를 통해 Next.js의 전반적인 흐름을 익혔고, 성능 최적화가 사용자 경험에 얼마나 큰 영향을 주는지 체감할 수 있었습니다.
