# 처음 보는 레거시 프로젝트, AI에게 코드 작성을 맡기기보다 분석을 맡겼다

새로운 프로젝트에 투입되면 가장 어려운 일은 코드를 작성하는 것이 아니라 **기존 코드를 이해하는 것**이다.

특히 운영 중인 서비스는 이미 여러 명의 개발자가 기능을 추가하고 수정하면서 구조가 복잡해져 있는 경우가 많다.

퓨쳐스콜레에서도 백오피스 프로젝트를 처음 맡았을 때 가장 먼저 한 일은 기능을 만드는 것이 아니라 "이 프로젝트가 어떻게 동작하는지"를 이해하는 것이었다.

당시 프로젝트는 Vue2와 Vue3가 함께 사용되고 있었고, 화면마다 상태 관리 방식도 조금씩 달랐다. 같은 역할을 하는 컴포넌트처럼 보여도 데이터가 전달되는 방식이 달랐고, API 호출 위치도 일정하지 않았다.

예전 같았으면 관련 파일을 하나씩 열어가며 흐름을 따라갔겠지만, 시간이 꽤 오래 걸렸을 것 같다.

그때부터 AI를 코드 생성 도구보다 **프로젝트를 분석하는 도구**로 활용하기 시작했다.

---

# 처음에는 AI에게 이런 질문을 했다

처음에는 단순하게

> 이 코드 설명해줘.

라고 물어봤다.

답은 틀리지 않았지만 크게 도움이 되지는 않았다.

컴포넌트 안에서 어떤 코드가 실행되는지는 설명해 줬지만,

- 왜 이런 구조인지
- 어디서 호출되는지
- 어떤 컴포넌트와 연결되는지

같은 내용은 알기 어려웠다.

결국 AI도 내가 주는 정보 안에서만 답할 수 있다는 걸 알게 됐다.

---

# 질문을 바꾸니 결과도 달라졌다

이후에는 질문 방식을 바꿨다.

예를 들어 회원 관리 화면을 분석할 때는

```
이 컴포넌트의 역할을 설명해줘.

props가 어디서 전달되는지

API는 어디에서 호출되는지

상태는 어디에서 변경되는지

다른 컴포넌트와 연결되는 부분까지 함께 정리해줘.
```

처럼 **코드가 아니라 흐름을 설명해 달라고** 요청했다.

그러자 단순한 코드 설명이 아니라

- 데이터 흐름
- 상태 변경 시점
- 컴포넌트 관계

를 한 번에 이해하기 쉬워졌다.

---

# 한 번에 많은 파일을 주지 않았다

처음에는 프로젝트 전체를 넣으면 더 좋은 답을 줄 거라고 생각했다.

하지만 실제로는 반대였다.

정보가 너무 많아지면 핵심을 놓치는 경우가 많았다.

그래서

App

↓

Page

↓

Container

↓

Component

↓

Hook

순서로 하나씩 분석했다.

덕분에 흐름을 따라가기가 훨씬 쉬웠다.

특히

"이 Hook은 어디에서 사용하는가"

같은 질문은 반복해서 많이 사용했다.

프로젝트 구조를 이해하는 속도가 눈에 띄게 빨라졌다.

---

# 리팩토링할 때도 먼저 분석을 시켰다

컴포넌트가 커졌다고 해서

바로

> 리팩토링해줘.

라고 하지는 않았다.

대신

```
이 컴포넌트의 책임을 나눠줘.

UI

비즈니스 로직

API

상태관리

관점에서 각각 어떤 역할인지 정리해줘.
```

처럼 먼저 분석을 요청했다.

생각보다

"왜 이렇게 작성됐는지"

를 이해하는 데 도움이 많이 됐다.

덕분에

무작정 Custom Hook으로 분리하거나

불필요하게 컴포넌트를 쪼개는 일도 줄어들었다.

---

# AI가 가장 도움이 됐던 순간

개인적으로 가장 만족했던 부분은

**처음 보는 코드에 대한 심리적 부담이 줄어든 것**이었다.

예전에는

"이 함수는 왜 있는 거지?"

"이 상태는 누가 변경하지?"

같은 질문을 해결하기 위해 여러 파일을 계속 오가야 했다.

지금은 AI에게

> 이 값이 변경되는 경로를 추적해줘.

라고 먼저 요청하고,

실제로 코드를 확인하면서 비교하는 방식으로 작업하고 있다.

답을 그대로 믿기보다는

**길을 안내받는 느낌**에 가까웠다.

---

# 결국 중요한 건 질문이었다

AI를 사용하면서 가장 크게 바뀐 건

코드를 작성하는 방식이 아니라

질문하는 방식이었다.

예전에는

"이 코드 만들어줘."

가 대부분이었다면,

지금은

- 이 구조가 적절한 이유는?

- 이 상태를 전역으로 둘 필요가 있을까?

- 더 단순한 구조는 없을까?

- 현재 구조의 문제점은 무엇일까?

같은 질문을 훨씬 많이 하게 됐다.

좋은 답은 좋은 질문에서 나온다는 말을 실감한 순간이었다.

---

# 마무리

AI 덕분에 개발을 더 잘하게 됐다고 말하기는 어렵다.

하지만 처음 보는 프로젝트를 이해하는 속도는 확실히 빨라졌다.

특히 운영 중인 서비스처럼 규모가 큰 프로젝트에서는 코드를 작성하는 시간보다 기존 구조를 이해하는 시간이 더 길다.

지금도 AI에게 코드를 대신 작성해 달라는 요청은 자주 하지 않는다.

대신 프로젝트를 분석하고, 구조를 이해하고, 놓친 부분이 없는지 함께 검토하는 과정에서 가장 많이 활용하고 있다.

결국 AI는 코드를 대신 작성해 주는 도구보다, **낯선 프로젝트를 조금 더 빠르게 이해할 수 있도록 도와주는 개발 파트너**에 가까웠다.
