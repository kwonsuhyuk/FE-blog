# Node.js란 무엇인가?

Node.js는 Chrome V8 JavaScript 엔진으로 빌드된 JavaScript 런타임입니다. 브라우저 밖에서도 JavaScript를 실행할 수 있게 해주며, 서버 사이드 개발에 널리 사용됩니다.

## 1. Node.js의 주요 특징

- **싱글 스레드 (Single Thread)**: 한 번에 하나의 작업만 수행하지만, 비동기 처리를 통해 효율적으로 동작합니다.
- **이벤트 기반 (Event-Driven)**: 이벤트가 발생할 때 미리 지정해둔 작업을 수행하는 방식입니다.
- **논블로킹 I/O (Non-blocking I/O)**: 이전 작업이 완료될 때까지 기다리지 않고 다음 작업을 수행하여 멈춤 없는 처리가 가능합니다.

## 2. HTTP Method와 서버 통신

서버와 데이터를 주고받을 때 사용하는 주요 방식들입니다.

- **GET**: 데이터를 **조회**할 때 사용합니다.
- **POST**: 새로운 데이터를 **생성**할 때 사용합니다.
- **PUT**: 기존 데이터를 **전체 수정(덮어쓰기)** 할 때 사용합니다.
- **PATCH**: 기존 데이터의 **일부만 수정**할 때 사용합니다.
- **DELETE**: 데이터를 **삭제**할 때 사용합니다.

## 3. 라이브러리와 프레임워크

- **모듈 (라이브러리)**: 특정 기능을 수행하는 코드의 묶음입니다. NPM(Node Package Manager)을 통해 관리됩니다.
- **프레임워크**: 개발을 더 편하게 하기 위해 자주 사용되는 라이브러리들을 모아둔 뼈대입니다. (예: Express.js, Nest.js)

## 4. 핵심 개념: 콜백 함수 (Callback Function)

JavaScript의 비동기 처리를 이해하는 데 필수적인 개념입니다. 함수의 매개변수로 다른 함수를 전달하여, 특정 작업이 끝난 뒤에 실행되도록 하는 방식입니다.

```javascript
// 콜백 함수의 간단한 예
function greet(name, callback) {
  console.log("Hello, " + name);
  callback();
}

greet("Suhyuk", function() {
  console.log("작업이 완료되었습니다!");
});
```

---

Node.js는 가볍고 효율적이어서 대규모 네트워크 애플리케이션을 구축하는 데 매우 적합합니다. JavaScript 하나로 프론트엔드부터 백엔드까지 개발할 수 있다는 점이 가장 큰 매력입니다.
